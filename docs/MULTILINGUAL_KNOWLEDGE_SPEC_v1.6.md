# 多言語Knowledge仕様 v1.6

## 目的

日本語を中心とする出品データを正本として維持しながら、日本語・英語・中国語・韓国語・ローマ字の質問を同じ商品へ結び付ける。

## データ原則

- `Master_Database`の日本語商品名、ASIN、SKU、在庫、価格を唯一の正本とする。
- 言語別に商品を複製しない。翻訳の古さによる在庫・価格・利益の不一致を防ぐ。
- `Search_Alias`は検索語とASINを結ぶ索引であり、`Approved=TRUE`だけを使用する。
- `Localized_Content`は回答表示用の名称・説明であり、`Approved=TRUE`だけを使用する。
- 翻訳が未承認または存在しない場合は、日本語の商品名を表示する。

## 対応言語

| Language | 用途 | 例 |
|---|---|---|
| JA | 日本語の質問・商品名 | 朝食シリアル |
| EN | 英語の検索別名・表示文 | breakfast cereal |
| ZH | 中国語の検索別名・表示文 | 早餐麦片 |
| KO | 韓国語の検索別名・表示文 | 아침 시리얼 |
| ROMAJI | 在日外国人等のローマ字入力 | choushoku shiriaru |

すべての例は同一ASINを指す。ローマ字は商品名のカナを自動変換した索引に加え、承認済みの別名も使用する。

## 検索と回答

1. 質問の言語を推定する。
2. 契約tenantの商品だけを取得する。
3. 同じtenant・ASINの承認済み別名と表示文を付加する。
4. 元の日本語、ローマ字、各言語の別名を横断して照合する。
5. 関連性の基準を満たす商品だけを最大3件返す。
6. 表示文が承認済みなら質問言語で表示し、なければ日本語へフォールバックする。

販売利益は順位に使用しない。根拠不足時は推薦しない。生の質問文はログへ保存せず、SHA-256ハッシュだけを保存する。

## SEO運用

`Multilingual_SEO`は商品ごとにローマ字・英語・中国語・韓国語の整備状況を採点し、未整備の言語を`Missing_Actions`へ表示する。スプレッドシートの「Project GATE → 多言語SEOを更新」で再集計する。

## 現段階の制限

- 機械翻訳を自動公開しない。Gemini等で下書きを作る場合も、人の承認後に`Approved=TRUE`とする。
- 同音異義語、ブランド固有表記、法規制表現は承認済み別名で補う。
- 購入後の言語別CVRは`KPI_Event_Log`で実測し、検索語や表示文を改善する。
