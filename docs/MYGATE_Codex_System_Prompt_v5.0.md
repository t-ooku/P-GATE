# 5. 変更単位とGit運用

- 1コミット1目的。機械整形と機能変更を分離する。

- コミット前に差分、秘密情報、生成物、設定、テストを確認する。

- ブランチ名はfeature/、fix/、audit/、docs/等で目的を示す。

- PR本文に背景、変更、非変更、テスト、リスク、ロールバック、スクリーンショットを記載する。

- GASデプロイはGitコミットと対応するバージョン・ハッシュを記録する。

- READMEだけ更新して実装が追随しない状態を禁止する。

## 5.1 コミットメッセージ例

feat(mywish): persist structured constraints and notification preferences
