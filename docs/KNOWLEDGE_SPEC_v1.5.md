# Project GATE Knowledge回答仕様 v1.5

## 目的

エンドユーザーの「欲しい」「知りたい」に対し、確認可能な商品情報だけを根拠として候補を最大3件返す。

## 回答フロー

1. 利用同意を確認する。
2. 契約IDからtenant、顧客種別、顧客IDを確定する。
3. 質問を正規化し、英数字語と日本語2文字単位へ分解する。
4. 同じtenantの商品だけを対象に照合する。
5. 関連性、情報充足、在庫から候補を採点する。
6. 関連性20%未満の商品を除外する。
7. 最大3件へ絞る。
8. 契約・競合・独占ポリシーで表示可否を判定する。
9. 許可された場合だけ候補と根拠を返す。

## ランキングに使う情報

- 質問と商品名・メーカー・ASIN・SKUの一致
- 商品名、メーカー、画像、Amazon URLの情報充足
- 在庫の有無

## ランキングに使わない情報

- セラー利益
- 契約料金
- 独占追加料金
- 広告出稿額

顧客契約は「表示してよいか」の判定にだけ使用し、エンドユーザーにとっての順位を買える仕組みにはしない。

## 根拠

各候補へ次を付与する。

- 一致した語句
- 情報充足スコア
- Master DatabaseのSource Hash
- 最終取込日時
- Confidence: `HIGH` / `MEDIUM` / `LOW`

## 回答しない条件

- 質問が2文字未満
- 検索可能な語句がない
- 関連性20%以上の商品がない
- 契約が無効またはカテゴリ外
- 競合同意・独占ポリシーに抵触

根拠不足時は商品を推測せず、「確認できる根拠が不足しているため推薦できない」と返す。

## プライバシー

- `Consent=true`が必須。
- 質問本文はシートへ保存しない。
- `Knowledge_Query_Log`には質問のSHA-256 Hashだけを記録する。
- tenantを越えた商品検索は行わない。

## 現段階の限界

v1.5は固定費ゼロで検証できる決定論的な日本語キーワード照合である。意味が同じ別表現や複雑な相談へ対応する意味検索・生成AIは未接続。

次段階では、この入出力仕様と中立性ルールを維持したまま、Gemini等の意味理解層とエンドユーザー画面を接続する。
